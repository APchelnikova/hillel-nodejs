# Що таке REPL ?

**REPL (Read-Eval-Print Loop)** — це інтерактивне середовище Node.js, яке дозволяє:
1.	Read (зчитування): Ввести JavaScript-код.
2.	Eval (обчислення): Виконати цей код.
3.	Print (виведення): Вивести результат.
4.	Loop (повторення): Повернутися до кроку зчитування.

Коли використовується REPL?
- Для швидкого тестування JavaScript-коду.
- Вивчення Node.js API.
- Дебагінг та експериментів із невеликими фрагментами коду.

### Як увійти в REPL?

1. Відкрийте термінал або командний рядок.
2. Введіть:
   node
3. Ви побачите такий інтерфейс:
    >
Це означає, що REPL готовий приймати ваш код.


### Основи роботи з REPL

1. Виконання простих операцій
Вводьте прості команди прямо в REPL: 
```js
> 2 + 2
4
> Math.max(10, 20, 30)
30
> 'Node.js'.toUpperCase()
'NODE.JS'
```

2. Робота зі змінними
У REPL можна оголошувати змінні та виконувати операції над ними:
```js
> const a = 5;
> const b = 10;
> a + b
15
> const message = 'Привіт, Node.js!';
> message.includes('Node.js')
true
```

3. Створення функцій
У REPL можна визначати функції:
```js
> function greet(name) {
... return `Привіт, ${name}!`;
... }
> greet('Світ')
'Привіт, Світ!'
> greet('Node.js')
'Привіт, Node.js!'
```

4. Використання багаторядкового коду
Якщо код складається з кількох рядків, завершіть кожен рядок комою, або введіть ..., щоб продовжити:
```js
> if (true) {
... console.log('Це працює!');
... }
Це працює!
```


### Використання команд REPL

REPL підтримує спеціальні команди, які починаються з крапки (.):

 .help — показує список доступних команд:
 - > .help
   .break    Вийти з багаторядкового вводу
   .clear    Очистити контекст REPL
   .exit     Вийти з REPL
   .load     Завантажити файл у REPL
   .save     Зберегти поточну сесію у файл

.exit — вихід із REPL:
> .exit

.clear — очищення поточного контексту:
> .clear
Всі змінні та функції видалені.


### Приклади роботи з REPL

1. Обчислення площі кола
```js
> const radius = 5;
> const area = Math.PI * radius ** 2;
> console.log('Площа кола:', area);
Площа кола: 78.53981633974483
```

2. Робота з масивами
```js
> const numbers = [1, 2, 3, 4, 5];
> numbers.map(x => x * 2);
[ 2, 4, 6, 8, 10 ]
> numbers.filter(x => x > 3);
[ 4, 5 ]
```


### Як зберегти або завантажити код у REPL?

#### Збереження коду:

1. Введіть в терміналі:
      .save ім'я_файлу.js
2. Наприклад:
   .save session.js
3. Усі команди з REPL збережуться у файл session.js.

#### Завантаження коду:

1.	Введіть:
      .load ім'я_файлу.js
2. 	Наприклад:
      .load session.js
3. Усі команди з файлу будуть виконані у REPL.


### Навіщо нам потрібен REPL?

1. Швидке тестування коду
- Інтерактивність REPL дозволяє швидко писати й перевіряти код без необхідності створювати файли.
- Можна миттєво перевіряти синтаксис функції, логіку циклу або результат обчислення.
```js
> const a = 10;
> const b = 20;
> a + b
30
```

2. Експерименти з функціями та API
- REPL дозволяє експериментувати з методами та API Node.js без тривалого налаштування проєкту.
- Можна спробувати методи модуля fs, path, або os на місці.
```js
> const os = require('os');
> os.platform()
'darwin' // або 'win32', залежно від ОС
```

3. Навчання через спроби та помилки
- Можна швидко побачити помилки, вчитися виправляти їх і розуміти, як працює код.
- Під час написання функції, якщо виникає помилка, REPL одразу вказує на проблему:
```js
> function add(a, b) {
... return a + b
... }
> add(5)
NaN // Не вистачає другого аргументу
```

4. Дебагінг
- REPL дозволяє відлагоджувати код, перевіряючи окремі частини програми.
- Можна перевіряти змінні чи виконання умов перед інтеграцією коду у більший проєкт.
```js
> const isEven = num => num % 2 === 0;
> isEven(4)
true
> isEven(5)
false
```

5. Робота з масивами та об’єктами
- Можна тренуватися виконувати операції з даними, маніпулювати масивами, об’єктами чи JSON.
```js
> const numbers = [1, 2, 3];
> numbers.map(x => x * 2);
[2, 4, 6]
> const user = { name: 'Катя', age: 20 };
> user.name
'Катя'
```

6. Знайомство з Node.js API
-  REPL дозволяє легко знайомитися з вбудованими модулями Node.js, такими як fs, os, або http.
```js
> const fs = require('fs');
> fs.readdirSync('.'); // Список файлів у поточній папці
[ 'index.js', 'app.js', 'test.js' ]
```
