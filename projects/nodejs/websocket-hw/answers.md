## 1. Чим цей чат принципово відрізняється від чату, зробленого на звичайних HTTP-запитах? Чому HTTP погано підходить для такої задачі?

- Чат на WebSocket відрізняється тим, що після встановлення з'єднання сервер і клієнт можуть надсилати повідомлення один одному 
в будь-який момент без створення нового запиту. 
- HTTP погано підходить для чату, тому що кожне повідомлення потребує нового HTTP-запиту та нової відповіді.

---

## 2. Що означає схема ws:// і чим відрізнялась би wss://?

- `ws://` — незахищене з'єднання (аналог http://).
- `wss://` — захищене, шифроване з'єднання через TLS (аналог https://).

---

## 3. У коді сервера повідомлення розсилається всім клієнтам (wss.clients.forEach). Що треба було б змінити, щоб надсилати його лише одному конкретному користувачу?

- Після підключення клієнта треба зберегти його `soket` та під час відправки повідомлення обрати конкретного клієнта замість foreach.
Наприклад:
```javascript
const clients = [];

wss.on('connection', (socket) => {
    console.log('Новий клієнт підключився');
    clients.push(socket);
    
    socket.on('message', (data) => {
        const text = data.toString();

        clients[2].send(text);
        socket.send(text);
    });

    socket.on('close', () => {
        console.log('Клієнт відключився');
    });
});
```

## 4. Чому в завданні 5 час додається на сервері, а не на клієнті? Який у цьому сенс?

- Час додається на сервері, тому що сервер є єдиним надійним джерелом часу.
- Це дозволяє уникнути помилок через неправильні налаштування годинника на пристроях користувачів.
